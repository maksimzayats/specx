## `services.read-no-writes-or-transactions`

Keep read services from writing data or owning transactions.

**Status:** Enabled by default.

### Why this exists

Read services may coordinate reads but must not mutate state or decide transaction lifetime.

### Failing example

```python
class OrderLookupService(BaseReadService):
    uow_manager: Injected[OrderUnitOfWorkManager]

    async def get(self, uow: OrderUnitOfWork, order_id: UUID) -> Order:
        await uow.orders.save(order)
        return order
```

### Passing example

```python
class OrderLookupService(BaseReadService):
    repository: OrderRepository

    async def get(self, order_id: UUID) -> Order:
        return await self.repository.get(order_id)
```

### Detection notes

Rejects injected UnitOfWorkManager fields, known effect calls such as commit, publish, send, and charge, and recognized repository mutators rooted in a UnitOfWork method argument.
