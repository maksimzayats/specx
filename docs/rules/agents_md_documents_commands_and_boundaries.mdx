## `agents-md.documents-commands-and-boundaries`

Keep generated-project agent guidance aligned with commands and boundaries.

**Status:** Enabled by default.

### Why this exists

Agents and contributors rely on AGENTS.md as an executable project contract, so it must agree with the actual Makefile and architecture in use.

### Failing example

```python
# AGENTS.md
# Run `make migration-check`.

# Makefile has no migration-check target and the project uses Alembic.
```

### Passing example

```python
# AGENTS.md
# Package lives under `src/acme`
# Run `make check`, `make lint`, and `make test`.
# Use cases return DTOs, not entities.
# `LoggingConfigurator` configures runtime logging. Do not inject loggers.

# Makefile defines check, lint, and test.
```

### Detection notes

Checks required stable phrases, conditionally required guidance based on used bases and Alembic, and every documented make target against the root Makefile.
