## `use-cases.return-dtos`

Require use cases to return DTOs instead of entities or raw values.

**Status:** Enabled by default.

### Why this exists

A stable DTO result keeps the application boundary independent from transport schemas and persistence entities.

### Failing example

```python
class CountOrdersQuery(BaseQuery):
    ...

class CountOrdersUseCase(BaseUseCase):
    async def execute(self, *, query: CountOrdersQuery) -> int:
        return 3
```

### Passing example

```python
class CountOrdersQuery(BaseQuery):
    ...

class CountOrdersUseCase(BaseUseCase):
    async def execute(self, *, query: CountOrdersQuery) -> OrderCountDTO:
        return OrderCountDTO(count=3)
```

### Detection notes

Checks `execute` return annotations on BaseUseCase descendants and requires each rendered type name to contain the case-sensitive substring `DTO`. It does not inspect BaseDTO ancestry.
