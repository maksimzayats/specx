## `services.effect-no-owned-transactions-or-delivery`

Keep effect services inside use-case-owned transaction boundaries.

**Status:** Enabled by default.

### Why this exists

Effect services perform focused effects inside a transaction boundary selected by the calling use case, and remain independent from transport code.

### Failing example

```python
from acme.delivery.fastapi.schemas.orders import OrderResponse

class NotificationService(BaseEffectService):
    async def send(self) -> None:
        async with self.uow_manager:
            ...
```

### Passing example

```python
class NotificationService(BaseEffectService):
    gateway: NotificationGateway

    async def send(self, message: MessageDTO) -> None:
        await self.gateway.send(message)
```

### Detection notes

Rejects delivery/framework imports, unit-of-work fields, and context-manager ownership in BaseEffectService descendants.
