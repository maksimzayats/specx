## `services.no-generic-base-service`

Reject generic `BaseService` in favor of effect-specific service bases.

**Status:** Enabled by default.

### Why this exists

A generic service base erases the most useful architectural distinction: whether behavior is deterministic, read-only, or effecting.

### Failing example

```python
class BaseService:
    ...

class PricingService(BaseService):
    ...
```

### Passing example

```python
class PricingService(BasePureService):
    ...
```

### Detection notes

Rejects class declarations named BaseService, imports of BaseService, and direct or qualified BaseService inheritance across project source.
