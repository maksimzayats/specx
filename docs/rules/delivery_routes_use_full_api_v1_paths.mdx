## `delivery.routes-use-full-api-v1-paths`

Require public business route registrations to use full `/api/v1/...` paths.

**Status:** Opt-in `fastapi` rule.

**Required project surface:** `delivery/fastapi`.

### Why this exists

Full versioned paths keep route ownership visible at registration sites and avoid hidden prefix composition.

### Failing example

```python
router = APIRouter(prefix="/api/v1")
router.add_api_route(path="/orders", endpoint=create_order, methods=["POST"])
```

### Passing example

```python
router.add_api_route(
    path="/api/v1/orders",
    endpoint=create_order,
    methods=["POST"],
)
router.add_api_route(path="/healthz", endpoint=health, methods=["GET"])
```

### Detection notes

FastAPI opt-in rule. Checks literal `path=` keywords on `add_api_route` calls below delivery controller packages. Positional or dynamic paths are reported; `/healthz` and `/readyz` are allowed exceptions.
