## `use-cases.inputs-are-local-commands-or-queries`

Require every use case to accept one same-file command or query input.

**Status:** Enabled by default.

### Why this exists

One explicit input object makes use-case calls evolvable and gives state-changing and read-only actions distinct names.

### Failing example

```python
class CreateOrderUseCase(BaseUseCase):
    async def execute(self, customer_id: UUID, total: Decimal) -> OrderDTO:
        ...
```

### Passing example

```python
class CreateOrderCommand(BaseCommand):
    customer_id: UUID
    total: Decimal

class CreateOrderUseCase(BaseUseCase):
    async def execute(self, command: CreateOrderCommand) -> OrderDTO:
        ...
```

### Detection notes

Requires one non-self execute parameter named command or query, annotated with a same-file BaseCommand or BaseQuery subclass whose kind matches the parameter name.
