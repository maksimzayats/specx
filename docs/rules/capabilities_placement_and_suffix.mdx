## `capabilities.placement-and-suffix`

Keep capability classes in capability packages with capability naming.

**Status:** Enabled by default.

### Why this exists

Capabilities are narrow injectable collaborators. Their package and suffix should make that role immediately visible.

### Failing example

```python
# core/orders/services/hash_helper.py
class PasswordHelper(BaseCapability):
    ...
```

### Passing example

```python
# core/orders/capabilities/password_hash.py
class PasswordHashCapability(BaseCapability):
    ...
```

### Detection notes

Places every BaseCapability descendant under a core `capabilities` package or `shared`. Only a direct BaseCapability subclass is required to end in `Capability`; the separate category-suffix rule handles narrower bases such as BaseClock.
