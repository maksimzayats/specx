# Agent-Led Setup Is the Setup Interface

modern-python-template will treat the prompt template given to an LLM agent as the setup authority for generated projects, instead of maintaining a deterministic wizard or answer-file contract as the primary interface. This is deliberate: project creators should remove unwanted capabilities from natural-language instructions and let the agent adapt the template, accepting some non-determinism as part of the setup model. Capabilities omitted from the prompt should be deleted from generated projects rather than left dormant or merely unconfigured, so generated projects carry only the features the project creator kept in the prompt. The existing setup wizard is legacy and should be retired rather than kept as a removable capability.
