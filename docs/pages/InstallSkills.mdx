<Title>Install agent skills</Title>
<Subtitle>Add the specx workflows to Codex and verify what was installed.</Subtitle>

## Install from GitHub

Run the installer from the project where the agent will work:

```sh
npx skills add maksimzayats/specx --skill '*' --agent codex -y
```

The command installs the catalog for the `codex` target. It does not add the `specx` Python package
to your application and does not change application code.

## Inspect a checkout before installing

Clone the repository when you want to review the skill text or test a branch:

```sh
git clone https://github.com/maksimzayats/specx.git
cd specx
npx skills add . --list --full-depth
```

Canonical skill definitions live under `skills/`. The tracked `.agents/skills/` directory is a local
discovery mirror maintained by this repository; do not copy both trees into an application.

## Install one workflow

Replace `'*'` with a skill name when the agent only needs a narrow workflow:

```sh
npx skills add maksimzayats/specx --skill specx-add-core-use-case --agent codex -y
```

## Use a skill

Name a skill in the request when you want its workflow applied explicitly:

```text
Use $specx-add-infrastructure-adapter to add the PostgreSQL order repository and unit of work.
Do not add the HTTP delivery layer yet.
```

You can also describe the work without naming a skill. Each skill has a trigger-oriented description
that lets the agent choose it from the task.

## Update the catalog

Run the install command again to update. Review skill diffs because they can change future agent
decisions and code structure.
